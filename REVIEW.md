## 一、这是什么项目（量化定位 — Fable）

**是量化，而且是当代最严谨的一派。** 从生态学/农学的"都试试"出发，诚实做下去，独立收敛到了 López de Prado 的现代量化反过拟合框架——机构与对冲基金用的就是这套。卖点不是"信号准"，而是**诚实分清真规律 vs 幻觉**。

### 量化方法论对照（我们 vs 专业文献）
| 专业标准方法 | 我们 | 状态 |
|---|---|---|
| Walk-forward 验证（Pardo，黄金标准） | 六折滚动 | ✅ |
| Purged + Embargo CV（López de Prado 2017，防时序泄漏） | factor_pruning/vol_model | ✅ |
| 块自助（重叠窗口正确显著性） | block_bootstrap | ✅ |
| 回测过拟合警惕（Bailey & LdP 2014：试几个配置就能凑高回测） | 逻辑回归/因子/波动全证伪 | ✅ |
| 硬基线 + 多重比较校正 | benchmark + BH 注释 | ✅ |
| **可升级**：Combinatorial Purged CV (CPCV) + Deflated Sharpe | 尚未 | ⬜ 下一级严谨度 |

**升级方向是验证方法（CPCV/Deflated Sharpe），不是加模型**（XGBoost 等花哨模型只会让回测更好看、样本外更差——这正是文献核心警告）。

- **旧 `renderEventStudyChart`（app.js）**：与新"事件影响一览"重复，且**保留了裸 p 值 + 方向红绿色（违反铁律）** → 删旧面板 + 对应 tab。【高·诚实性回退】

- **`--full` ML 簇**（advanced/multivariate，1370 行）：CI 永不跑，multivariate.json 冻结于 6/9 却以"实时图表"展示 → CI 收盘后加跑 --full，或前端标"快照"。【高·诚实性漏洞】

### 数据健壮性（最影响"前向证据"）
- **yfinance 无缓存兜底**（fetch_data._get_close）：Yahoo 限流→列静默消失（FRED 有兜底，Yahoo 没有）→ 加同款"失败读缓存"回退。【高】
- **verify_output 不验列完整性**：掉一列照样过门禁发布 → 加"关键列存在且末值非 NaN"检查。【高】

### 架构 / 性能
- app.js 3593 行 → 最小 ESM 拆分（先抽 lib.js，再按视图）+ IntersectionObserver 懒渲染。【中】
- 两遍 build_signals → 拆 compute/assemble（顺带消灭 light/full 信号差异）。【中】
- signals.json 218KB → 历史行只留 {prob,tier}、日历字段前端算，可再瘦 ~40%。【中】
- 日历因子双实现（生产硬编码 vs wf 学习值）仍未统一（ROADMAP P2-1）。【中】
