## 1. 现状（已建好的诚实地基，别推倒）

`autodiscovery.py`(Phase 1b) 已经是一套**严谨的诚实管线**，不是"仅展示层"：
- `candidate_space.py`：**预注册、有限、可枚举** 42 个候选（日历15 + 反弹12 + 因子15），`N_DECLARED` 写死、test 对账、扩展须改 spec 走双审。
- `compute_results`：每候选路由到**真统计**（placebo 置换 / 块自助 / 因子分段 p），出 `{p, recent_p, recent_powered}`。
- `quality_gate.adjudicate`：**双栏 BY-FDR（族内 + 跨族）+ 三态裁决**（存活 / 已淡 / 检验力不足）。
- `autodiscovery_log.csv`：**append-only 每日裁决快照**（被 CI 持久化）= 闭环的"记忆"/前向史。
- 防 p-hacking 护栏：固定种子 · **全部候选进 FDR 分母（禁预筛）** · 候选预注册。

## 2. ⚠️ 核心张力（必须先拍板，否则会建出一台 p-hacking 机器）

"**自生长 / 自动发现**"字面意思 = 系统**自动生成**新假设去测。
但本项目的命门正是：**候选必须预先声明、人工评审、有限可枚举**。自动无界生成候选 = forking paths = 数据钓鱼 = 我们一直反对的东西。

**两者直接冲突。** 所以闭环**不能**是"自动发明并检验新规律"。

→ **诚实的"闭环"必须把'自动'放在下游（验证/晋升/衰减），把'人工把关'留在上游（候选注册）。**

## 3. 诚实的「闭环」定义（建议方案）

候选**仍然预注册**（上游人工 + 双审，命门不动）。自动化的是**下游的前向确认 + 知识库自生长**：

```
预注册候选(人工·锁定)
   → 每日 真统计 + 双栏FDR + 三态   [已建好]
   → 【新】门4：样本外(OOS)确认 —— 只认"声明/发现日之后"的数据是否仍成立
   → 【新】衰减/稳定度追踪 —— 读 autodiscovery_log 前向史，看裁决是否稳定(非闪烁)
   → 【新】自动晋升/降级 → evidence 知识库(evidence_ledger)
        · 晋升门槛(全部满足才进库)：跨族FDR存活 ∧ 现代段有检验力 ∧ OOS不翻盘 ∧ 前向裁决稳定 N 天
        · 自动降级：曾入库但 OOS 翻盘 / 持续 faded → 标"已淡出"，不删历史(append-only)
```

**"自生长"= 知识库随证据前向累积而生长/收缩**，不是候选空间自动膨胀。这守得住命门、又真有"自己长"的动态。

### 自动**提案**层（用户已选纳入·2026-06-26）
系统在**预声明的元模式**内自动列出"候选提案"（如反弹族 pctl×hold 网格里尚未注册的格点、日历族未覆盖的既有学术先验），但 **提案 ≠ 入FDR**——必须人工 append-only 注册 + 双审才进分母。即"机器提议、人类把关"，保留命门、又减人工枚举负担。
**关键护栏**：提案只能来自**有限、预声明的元空间**（不是无界网格）；提案**不算 p、不进分母、不上前端"战绩"**，只进一个 `proposals` 待审列表；人工注册时记 append-only（日期/理由）。否则"自动提案"会从后门变成钓鱼。

## 4. 分阶段（每阶段 = 独立可验证切片 + 各自审）

- **P-A 门4 OOS 引擎**（最核心·判断最密集）：对每个候选定义"声明/发现锚点日"，只用锚点后数据复算 p/方向是否仍成立。**判断点**：锚点日怎么定（候选首次进 candidate_space 的 commit 日？还是数据起点？）；OOS 样本不足时必须返回"未到可判"而非假阴/假阳。→ 单独建 + 全新 Opus 审。
- **P-B 衰减/稳定度**：读 `autodiscovery_log.csv` 前向史，给每候选算"裁决稳定度"（如近 N 个交易日裁决一致比例）。纯描述、不引新统计。
- **P-C 晋升/降级 → 知识库**：把 P-A+P-B 的结论接到 `evidence_ledger`，按门槛自动进/出库（append-only，绝不改历史行）。
- **P-D 前端**：`self_growing.html` 展示"知识库生长史"（哪条何时晋升/降级 + OOS 战绩）。展示层，subagent 可做、我验。
- **P-E 自动提案层**（用户已选）：在预声明元空间内枚举"未注册格点"→ 写 `proposals` 待审列表（不算 p、不进分母、不进前端战绩）。人工注册仍走 candidate_space + 双审。**判断点**：元空间边界划在哪（必须有限、可声明）；如何防它悄悄变无界。

## 5. 护栏（红线，建造时逐条守）

1. 候选空间**不自动膨胀**——`N_DECLARED` 仍写死、扩展走双审。
2. **OOS 锚点一旦定下不可回改**（否则就是事后挪靶 = 作弊）；锚点入 append-only 记录。
3. 晋升/降级账本 **append-only**，绝不改历史行（同 prediction_log 红线）。
4. OOS/衰减**样本不足 → "未到可判"**，绝不凑结论。
5. 全部仍走 `verify_output` / `pytest` 门禁；新统计上线前 walk-forward/CPCV 自证（若引入模型）。
6. 公开标不确定性 + "存活≠未来重演≠可交易"。
