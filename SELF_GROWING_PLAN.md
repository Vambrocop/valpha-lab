## 1. 现状（已建好的诚实地基，别推倒）

`autodiscovery.py`(Phase 1b) 已经是一套**严谨的诚实管线**，不是"仅展示层"：
- `candidate_space.py`：**预注册、有限、可枚举** 42 个候选（日历15 + 反弹12 + 因子15），`N_DECLARED` 写死、test 对账、扩展须改 spec 走双审。
- `compute_results`：每候选路由到**真统计**（placebo 置换 / 块自助 / 因子分段 p），出 `{p, recent_p, recent_powered}`。
- `quality_gate.adjudicate`：**双栏 BY-FDR（族内 + 跨族）+ 三态裁决**（存活 / 已淡 / 检验力不足）。
- `autodiscovery_log.csv`：**append-only 每日裁决快照**（被 CI 持久化）= 闭环的"记忆"/前向史。
- 防 p-hacking 护栏：固定种子 · **全部候选进 FDR 分母（禁预筛）** · 候选预注册。

## 2. ⚠️ 核心张力（必须先拍板，否则会建出一台 p-hacking 机器）

"**自生长 / 自动发现**"字面意思 = 系统**自动生成**新假设去测。
但本项目的命门正是：**候选必须预先声明、人工评审、有限可枚举**。自动无界生成候选 = forking paths = 数据钓鱼 = 我们一直反对的东西。

**两者直接冲突。** 所以闭环**不能**是"自动发明并检验新规律"。

→ **诚实的"闭环"必须把'自动'放在下游（验证/晋升/衰减），把'人工把关'留在上游（候选注册）。**

## 3. 诚实的「闭环」定义（建议方案）

候选**仍然预注册**（上游人工 + 双审，命门不动）。自动化的是**下游的前向确认 + 知识库自生长**：

```
预注册候选(人工·锁定)
   → 每日 真统计 + 双栏FDR + 三态   [已建好]
   → 【新】门4：样本外(OOS)确认 —— 只认"声明/发现日之后"的数据是否仍成立
   → 【新】衰减/稳定度追踪 —— 读 autodiscovery_log 前向史，看裁决是否稳定(非闪烁)
   → 【新】自动晋升/降级 → evidence 知识库(evidence_ledger)
        · 晋升门槛(全部满足才进库)：跨族FDR存活 ∧ 现代段有检验力 ∧ OOS不翻盘 ∧ 前向裁决稳定 N 天
        · 自动降级：曾入库但 OOS 翻盘 / 持续 faded → 标"已淡出"，不删历史(append-only)
```

**"自生长"= 知识库随证据前向累积而生长/收缩**，不是候选空间自动膨胀。这守得住命门、又真有"自己长"的动态。

### 自动**提案**层（用户已选纳入·2026-06-26）
系统在**预声明的元模式**内自动列出"候选提案"（如反弹族 pctl×hold 网格里尚未注册的格点、日历族未覆盖的既有学术先验），但 **提案 ≠ 入FDR**——必须人工 append-only 注册 + 双审才进分母。即"机器提议、人类把关"，保留命门、又减人工枚举负担。
**关键护栏**：提案只能来自**有限、预声明的元空间**（不是无界网格）；提案**不算 p、不进分母、不上前端"战绩"**，只进一个 `proposals` 待审列表；人工注册时记 append-only（日期/理由）。否则"自动提案"会从后门变成钓鱼。

## 4. 分阶段（每阶段 = 独立可验证切片 + 各自审）

- **P-A 门4 OOS 引擎**（最核心·判断最密集）：对每个候选定义"声明/发现锚点日"，只用锚点后数据复算 p/方向是否仍成立。**判断点**：锚点日怎么定（候选首次进 candidate_space 的 commit 日？还是数据起点？）；OOS 样本不足时必须返回"未到可判"而非假阴/假阳。→ 单独建 + 全新 Opus 审。
- **P-B 衰减/稳定度**：读 `autodiscovery_log.csv` 前向史，给每候选算"裁决稳定度"（如近 N 个交易日裁决一致比例）。纯描述、不引新统计。
- **P-C 晋升/降级 → 知识库**：把 P-A+P-B 的结论接到 `evidence_ledger`，按门槛自动进/出库（append-only，绝不改历史行）。
- **P-D 前端**：`self_growing.html` 展示"知识库生长史"（哪条何时晋升/降级 + OOS 战绩）。展示层，subagent 可做、我验。
- **P-E 自动提案层**（用户已选）：在预声明元空间内枚举"未注册格点"→ 写 `proposals` 待审列表（不算 p、不进分母、不进前端战绩）。人工注册仍走 candidate_space + 双审。**判断点**：元空间边界划在哪（必须有限、可声明）；如何防它悄悄变无界。

## 5. 护栏（红线，建造时逐条守）

1. 候选空间**不自动膨胀**——`N_DECLARED` 仍写死、扩展走双审。
2. **OOS 锚点一旦定下不可回改**（否则就是事后挪靶 = 作弊）；锚点入 append-only 记录。
3. 晋升/降级账本 **append-only**，绝不改历史行（同 prediction_log 红线）。
4. OOS/衰减**样本不足 → "未到可判"**，绝不凑结论。
5. 全部仍走 `verify_output` / `pytest` 门禁；新统计上线前 walk-forward/CPCV 自证（若引入模型）。
6. 公开标不确定性 + "存活≠未来重演≠可交易"。

**裁决：方向 SOUND-TO-BUILD（"自动只在下游、候选仍人工预注册"真守住命门），但 P-A 开建前必须钉死下列口径；P-E 推迟。** Opus 挑出 4 个真洞，均采纳：

锚点 = 候选的 **append-only 注册日**，存进新建不可变登记簿 `candidate_registry.csv`(`candidate_id, declared_date, reason, reviewer`)。
**妙处**：`candidate_id` = params 哈希 → 改参数 = 换 id = 被迫新注册 + 更晚锚点 → **物理上没法偷偷挪靶**。
旧 42 候选无注册日 → 锚点 = **计划采纳日(今天)前向累积**（干净不可争），**不回填数据起点**（那是假长 OOS）。
门4 窗口 = 每候选 `idx > anchor[cid]`，与全局 `RECENT_CUT` **正交、单列报**；"未到可判" = 一等公民，绝不借裁决。最小 OOS 检验力沿用 `MIN_GROUP_N`/≥200，不足 → "未到可判"。

**② 稳定度 re-scope（P0·主脑担心的那洞，确认成立）**：每日日志是**同一累积数据**的快照 → "稳定 N 天" ≈ 一个检验重读 N 次（自相关 → 假稳定），**不是 N 次独立确认**。
→ P-B 改成**翻烧饼/防闪烁探测器**（只抓在 FDR 边界来回横跳的脆弱候选），明面标"裁决 N 天没变，但底层样本只长了 X%"。**绑定证据是 OOS（门4），不是自相关稳定度**。"N 交易日"按真实市场 session 数，不按日志行数（`append_daily_log` 按日历日去重，非交易日也可能落行）。

**③ 晋升 = 新的多重比较面（P1）**：42 候选 × 每天问"今天进库没" = 二次选择层，现有 FDR 没覆盖 → 边界候选总会某天侥幸全绿被晋升。
→ 晋升绑定 **OOS 在实质增长的 post-anchor 窗上持续成立**（非自相关全样本）；加**滞回（promote/demote 不同阈值）**防 chatter；**append-only 单调晋升**（进库后除非 OOS 翻盘才降级），**不对晋升再跑 FDR**（会双罚）——审计痕迹 = 诚实机制。每条进/出库行自带 `anchor_date/oos_n/oos_p/cross_fdr_q/trigger`（可审不必重算）。

**④ 自动提案 P-E（P1·最高风险后门）**：'未注册格点' = 变相无界阈值扫；且"机器 surface 赢家 → 人类'独立'注册" = forking-paths 洗白（人注册哪个由数据驱动）。
→ **P-E 推迟到最后，v1 不含它**（先把核心环 P-A/B/C/D 跑通证明可靠）。建时硬门槛：元空间必须有限 + `N_META_DECLARED` 声明 + test 对账；提案**按数据无关顺序排、绝不显示任何 p/统计**；注册理由必须是**先于数据的先验**（非"测得好"）。达不到就别建。

**N_DECLARED 变更（P2）**：每日志行存 `n_declared` + 候选集哈希；只在同分母下比纵向；N 变处标"前后不可直接比"。
**"翻盘"定义（P3）**：OOS 方向反号 **OR** bootstrap p>0.20（故意宽 → 只在真失败才降级，喂①的滞回）。
**可复现（P2）**：晋升行存 `oos_p/oos_n`，决策可审不必快照数据。

## 10. 门4 OOS 引擎 — 实现设计（2026-06-26 算清·开建前锁定·供 Opus 审实现）

**复用 autodiscovery 计算 + `floor` 开关只看锚后（各族语义不同·命门）：**
- `_calendar`：`floor` 过滤**输入** `ret`（日历无前看依赖，对）。
- `_rebound`：阈值 = **全样本**百分位（规则定义于全数据）→ 只把 `(sel,y)` 过滤到锚后，**不能** floor 输入（否则阈值变）。
- `_regime`：`ma50/200` 用**全 px** 算 → 只把 `(cond,fwd)` 过滤到锚后，**不能** floor 输入（否则均线重启）。
- 小样本边缘：任一组 `< MIN_GROUP_N` → 返回 None → **"未到可判"**（防空组崩 / 假 p）。

**`oos_gate.py`**（每候选 anchor = `candidate_registry.declared_date`）：
- 未到可判：锚后触发组 `< MIN_OOS_N`
- confirmed：方向同 ∧ `oos_p < 0.10`
- overturned：方向反号 **OR** `oos_p > 0.20`（滞回：confirm/overturn 不同阈值，防 chatter）
- 持中（0.10–0.20 同向）：不动

**晋升门（全满足）**：跨族 FDR 存活 ∧ 现代有检验力 ∧ confirmed ∧ 不在库 → promote。
**降级**：在库 ∧ overturned → demote。**append-only 单调**（进库后只 overturn 才动）；**绝不对晋升再跑 FDR**（双罚）；审计痕迹 = 诚实机制。
`kb_ledger.csv`(append-only)：`date, candidate_id, action, anchor_date, oos_n, oos_p, oos_sign, trigger`。

**测试**：合成"锚在过去 + 锚后数据 成立/翻盘/不足" → promote/demote/未到可判/单调/滞回 各断言。

## 12. P-E 自动提案层 — 实现设计规格（2026-07-01 写·开建前锁定·步骤①·供审计划②挑洞）

> 状态：**规格草案，未动代码**。P-A/B/C/D 已建成上线（§11）；P-E 是自生长最后一块、CLAUDE.md 明标"最高风险后门"。
> 按六步：①本规格 → ②独立审规格(全新 Opus + 用户挑洞) → ③建 → ④双审实现 → ⑤修 → ⑥验证提交。**规格审过才动代码。**

### 12.1 P-E 是什么 / 不是什么（先钉死语义，防它变钓鱼机）
- **是**：一张**覆盖度地图**。在**预声明、有限**的"元空间"里，列出"哪些格点已注册、哪些尚未注册"，把未注册的写进 `proposals` 待人工评审清单。
- **不是**：不 surface 赢家、不测、不算、**完全不碰任何市场数据**。P-E 是**纯组合运算**：`proposals = 元空间 − 已注册`。零统计、零 p、零 peek。
- **关键推论**：P-E 的风险**全在设计**（元空间是否诚实有界），**不在实现**（建造是机械的、无数据）。→ **规格审是关键闸门。**
- **价值**：减少"下一个该注册啥"的人工枚举/检索负担 + 让候选空间的覆盖缺口可见。**刻意惰性**——它只提议，不推动。

### 12.2 元空间定义（§7④ 铁律：必须有限 + `N_META_DECLARED` 声明 + test 对账 + 先验驱动）
命门现状（§1）：`candidate_space.py` 已是 80 候选的**完整枚举**——当前声明族内**没有**未注册格点。故 P-E 的"未注册格点"只能来自一个**比当前注册集更大的、预声明的元空间超集**。这个超集怎么划，就是 P-E 的全部诚实风险。

**硬约束**：元空间必须 (a) 有限可枚举；(b) `N_META_DECLARED` 写死 + test 对账；(c) **⊇ 当前全部已注册候选**（否则元空间划错了）；(d) 每个格点都对应一个**先于数据的具名先验/文献**（不是"参数网格随便撒点"）。

**候选元空间（两方案·请审②拍板）：**

- **方案 A（保守·推荐）：只列"具名+可引用"的先验缺口，不扩参数网格。**
  - **日历**：元空间 = 已注册效应 + 一份**具名学术先验短清单**（每条带文献），如 `monday_effect`(周一弱·French 1980)、`tax_loss_selling`(12月末小盘·税损抛售)等**确有文献**的未注册效应。日历空间近饱和 → 提案少（诚实地显示"多数已覆盖"）。
  - **体制**：加 `death_cross`(50<200日线·金叉的看跌镜像·文献对称先验) × 2 指数。
  - **反弹/因子**：**不扩网格**——网格扩张=阈值扫风险(§7④)，声明为"在已注册分辨率上视作完整；要扩须走 candidate_space 新 spec，不经 P-E"。
  - **好处**：每个提案都是**具名+带引用**的先验 → 注册天然先验驱动、无"任意阈值扫"向量。**最少后门。**

- **方案 B（含网格·原 §3 举例）：A + 反弹族有界网格缺口。**
  - 反弹元空间 `pctl ∈ {1,5,10,25}`(+25=四分位约定阈)、`hold ∈ {1,5,10,20}`(+10=两周·+20=一月) × 2 指数 = 32；已注册 12 → 提案 20。
  - **风险**：即便有界，"提案里列了 p25_h20"仍可能**诱导**无真先验的注册（Opus §7④ 警告的 forking-paths 洗白）。
  - **缓释**：提案零统计 + 注册闸门(先验+双审)不变 → 单凭提案清单不足以注册。但"诱导"效应真实存在。

  **主脑推荐 = 方案 A**（每提案具名带引用，彻底掐掉阈值扫向量；网格扩张若真要，走 candidate_space 正规双审更诚实）。请审②裁。

### 12.3 提案生成算法（纯组合·数据无关）
1. `meta_space()`：枚举元空间全部格点（每格 = {family, params, prior_name, prior_cite}）；`assert len(meta_space()) == N_META_DECLARED`。
2. `registered = {c.candidate_id for c in candidate_space.enumerate_candidates()}`（用同一 `_cid` 哈希，保证对齐）。
3. `assert 每个 registered 候选都能在 meta_space 里找到`（元空间 ⊇ 注册集，否则元空间划错、fail）。
4. `proposals = [g for g in meta_space() if _cid(g.family, g.params) not in registered]`。
5. **排序 = 数据无关**（按 `(family, prior_name, 参数字典序)` 稳定排）；**绝不**按任何统计/近期表现排。
6. 写 `proposals.json`（仅惰性字段，见 12.4）。**不 append 任何账本**（提案不是前向计分对象·非战绩）。

### 12.4 输出结构（`proposals.json`·惰性字段·零数据派生）
```json
{ "generated": "...Z", "n_meta_declared": <int>, "n_registered_in_meta": <int>, "n_proposals": <int>,
  "proposals": [ {"family": "...", "key_hint": "...", "params": {...}, "prior_name": "...", "prior_cite": "...",
                  "how_to_register": "改 candidate_space + append candidate_registry + 双审(理由须为先验非'测得好')"} ],
  "caveat": "机器只列'预声明元空间里尚未注册的格点'·不测不算不 peek·非战绩·非推荐。注册仍须人工写先验理由 + append-only + 双审。"}
```
**严禁字段**：任何 p / up_pct / recent_p / 排名分 / 任何数据派生量。verify_output 加断言：`proposals.json` 里**不得出现** `p`/`up_pct`/`recent_p`/`score` 等统计键（有=拒绝发布）。

### 12.5 护栏映射（§7④ 逐条 → 代码/测试如何强制）
| §7④ 护栏 | 强制手段 |
|---|---|
| 元空间有限 + 声明 + 对账 | `N_META_DECLARED` 写死；test `len(meta_space())==N_META_DECLARED` |
| 元空间 ⊇ 注册集 | test：每个 registered 候选都在 meta_space 里 |
| 提案零统计、数据无关排序 | 代码不 import 任何统计/数据模块；verify_output 断言无统计键；test 断言排序稳定且不依赖数据 |
| 不进 FDR 分母 / 不上战绩 | P-E 不碰 `autodiscovery`/`quality_gate`/`kb_ledger`；proposals 独立 json，前端单独区块标"待审·非战绩" |
| 注册须先验+双审 | P-E **不自动注册**；`how_to_register` 明写须走 candidate_space+registry+双审+先验理由 |
| append-only 命门不变 | P-E 只读 candidate_space/registry，不写它们 |

### 12.7 测试规格（`test_proposals.py`）
- `len(meta_space()) == N_META_DECLARED`（对账·漂移即 fail）。
- 元空间 ⊇ 注册集（每个 registered candidate 可在 meta 找到）。
- `proposals == meta − registered`（数量 + 具体格点）。
- 排序稳定且**数据无关**（同输入同序·不 import 数据）。
- 输出**无任何统计键**（p/up_pct/recent_p/score 都不在）。
- caveat 含"不测不算/非战绩/注册须先验+双审"诚实框。
- 优雅降级：candidate_space/registry 读失败 → 空 proposals + 退0。
