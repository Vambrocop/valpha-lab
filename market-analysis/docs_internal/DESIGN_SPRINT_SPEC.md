# DESIGN_SPRINT_SPEC — "不够高级"设计冲刺提案

起草:Fable 5 主脑,2026-07-09。状态:**待用户拍板,未开工**。
用户原话:"最大的问题还是看上去不够高级,很多没有数据呈现……更多图和互动、设计、点击、整个页面设计"。

## 0. 诊断:为什么"看上去不够高级"

看了半年这个站,我的判断是:**统计内核是机构级的,但呈现层还停在"工程师给自己看"的阶段**。
具体五个病灶(按对观感的伤害排序):

1. **数字裸奔,没有上下文** — 首页/仪表盘满是大数字(倾向分、VIX、概率),但没有"它最近怎么走的"。
   高级感的第一来源是**每个数字旁边有一条微趋势**(sparkline)——Bloomberg/Robinhood 的质感一半来自这个。
2. **图表密度太低** — Plotly 已本地化却用得省:dashboard 大部分面板是文字+表格。数据都算出来了
   (composite_log 有每日分数史、market_regime 有各成分史、40日概率日历有逐日值),就是没画。
3. **交互是"跳转式"不是"探索式"** — 点击基本只用来切页;没有 hover 细节、没有点开下钻、
   没有图表时间范围切换(7D/1M/6M/1Y)。用户的"点击"欲望没有落点。
4. **视觉层级平** — 所有卡片同一重量、同一边框、同一字号,首屏没有"一眼该看哪"的引导;
   数字排版没有统一 tabular-nums/字号阶梯,大小写混排显得散。
5. **页面是"死"的** — 无加载骨架(loading 是文字"加载中")、无数字滚动、无状态变化的微动效。
   高级感的另一半来自 150–250ms 的克制过渡,让页面感觉"活着且贵"。

## 1. 硬约束(不可破)

- **中国访客**:零境外 CDN、零外部字体——所有方案必须自托管/内联(inline SVG/CSS 动画/已有本地 Plotly)。
- **诚实红线**:装饰绝不制造假信号感——颜色只跟数据语义走(绿涨红跌紫品牌灰中性);
  校准平坦时的"全中性"绝不用视觉手段暗示决断(867217d 的结论式抬头是范式)。
- **静态站**:GitHub Pages,无后端;所有交互纯前端。
- **初学者中文用户**:大白话不牺牲;术语 hover 注解沿 vp_gloss 扩展。
- **性能**:sparkline 用内联 SVG(~50 行零依赖),不为微图引第二个图表库。

## 2. 冲刺分层(D1–D5,各自独立可验收;建议先 D1+D2 一轮见效)

**D1 视觉系统统一(CSS 为主·低风险·半天级)**
- 字号阶梯(display/number/body/caption 四档)+ 全站数字 `font-variant-numeric:tabular-nums`;
- 卡片三级重量:hero(带渐变边)/常规/工具项——现在 index 已有雏形(.tool-feat),推广到 dashboard;
- 间距刻度统一(4/8/12/16/24);语义色纪律成文(哪些元素允许绿/红);
- 验收:两页截图对比 + interaction_audit 无回归。

**D2 微图表层(sparkline·最高性价比·1 天级)**
- 新建 `vp_spark.js`(内联 SVG 折线/柱,~60 行,零依赖):`vpSpark(el, values, {color, fill})`;
- 落点:①首页 hero 倾向分旁 30 日走势 ②market_regime 每个成分旁 90 日微线
  ③composite 卡 30 日分数史 ④quotes 卡盘中走势(已有分时数据) ⑤观察台每条存活规律的近月应期带;
- 数据全部已存在(composite_log/market_regime history/quotes),纯呈现,零统计新口径;
- 验收:playwright 截图 + 移动端不溢出。

**D3 交互层(探索式点击·1–2 天级)**
- 40 日概率日历升级成**热力横带**(hover 出当日细节:概率/档位/宏观标签);
- 个股表行点击 → 展开该票 checkup 摘要卡(数据已在 stock_checkup.json);
- Plotly 图统一加范围切换(7D/1M/6M/1Y,restyle 不重拉数据);
- vp_gloss hover 注解覆盖率从当前扩到全部术语首次出现处;
- 验收:interaction_audit(它就是为这个建的)+ 每个新交互的死按钮检查。

**D4 动效层(克制·半天级)**
- 骨架屏替代"加载中"文字(CSS shimmer);数字首屏 count-up(300ms,prefers-reduced-motion 关);
- 面板切换/hover 过渡统一 150–250ms;盘中报价刷新时轻微 pulse;
- 红线:动效绝不用于强调某个方向判断(不许给"看涨"加闪光)。

**D5 首页信息架构(需用户参与·单独一轮)**
- 首屏改成"三个数字讲今天":倾向仪表(已有)+ 今日应期存活规律数 + 40日前瞻一句话结论;
- 工具网格按"看结论/查数据/玩出格"三组重排(grp-hd 已有机制);
- 这轮动 IA,**必须先出线框给用户看再动手**。

## 3. 工作流约定

- 每个 D = 独立六步(spec→审spec→Sonnet建→审→修→验),UI 改动必跑 interaction_audit + 移动端截图;
- D2 的 vp_spark.js 进 check_docs_mirror 守卫(vp_*.js 前缀自动纳入,e3d57a5 已泛化);
- 顺序建议:**D1+D2 先做**(观感提升最大/风险最小),D3 次之,D4 点缀,D5 最后且要线框确认。

## 4. 明确不做

前端框架/构建链(React/Vue/Tailwind)——违背同源自托管与零依赖纪律;外部字体;
任何"让数据显得更有信号"的视觉夸张;为动效引 JS 动画库(CSS 够用)。
