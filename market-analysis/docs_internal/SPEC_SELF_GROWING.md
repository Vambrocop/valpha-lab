# SPEC — v2.0 自生长引擎（自动发现规律 → 诚实质量门 → 登记簿/坟场 → 度量）

> 六步协议 ①写计划。**判断密集（公开统计结论）→ 建后双审（Fable + 全新 Opus/Codex）**。
> 写于 2026-06-19。这是全项目**统计上最危险**的功能，spec 的全部价值在「不让它变成 p-hacking 机器」。

## 一句话
让系统**自己**从一个**预先声明、有限**的候选空间里，把每个候选**过一遍现有诚实门**（placebo 分段 + 跨族 FDR + walk-forward 样本外 + Deflated Sharpe + conformal），
**活下来的进登记簿（真规律）、死的进坟场（假规律·带死因）**，并**长期复测**——衰减了就从登记簿移回坟场。知识库**诚实地自动生长**。

## 🔴🔴 中心危险（建造前必须内化，不是可选项）
**自动生成 + 批量检验 = 数据挖掘 / p-hacking。** 测 1000 个随机候选，α=0.05 下**期望 50 个"显著"全是假的**。
naive 实现会让本项目**反噬自己的魂**（"拆穿假规律"变成"批量制造假规律"）。所以：

1. **候选空间必须预先声明、可枚举、有限**（N 已知）。**禁止无界钓鱼**（不许"试遍所有变换直到显著"）。N 写死在代码 + 文档，每次扩 N 都要记录（append-only）。
2. **多重检验更正跨【整个家族】算，不是逐候选**。用 Benjamini-**Yekutieli**（容许相关，比 BH 保守）控 FDR；外加 **Deflated Sharpe Ratio**（按 trials 数 N 缩水，直接惩罚"试了很多次"）。
3. **样本外是硬门**：walk-forward / CPCV 块自助，候选必须**样本外**也过，不只全样本。
4. **诚实账单常驻**：页面永远显示「本轮测了 N 个候选，期望假阳性 ≈ N×α_effective，实际入选 M 个」——让人一眼看出信噪。
5. **预注册**：候选空间 + 门槛 + 更正方法**先写死、后跑**，不许看了结果再调门槛（HARKing 禁止）。

> 若实现中想"放宽门让多几个通过"——**停下、别做、报告**。门是这功能的全部意义，松了就废了。

## 候选空间（v2.0 首版：小而预声明）
**有限、可枚举的一个家族**（N 几十量级，不是无穷）。首版只收**已有数据能算、且本就在测的**：
- 日历族：星期几 × {纳指, 标普}、月份、节假日前后（placebo 已在测这些）。
- 超跌反弹族：阈值 ∈ {第1,5,10 百分位} × 持有 {1,5 日}。
- 简单因子族：现有 factor_audit 里的因子 × {全样本, 现代段}（#5 分段透镜的产物）。
**不收**：自由参数扫描、新因子、深度模型（违反「不做的事」）。N 固定、写在 `candidate_space.py` 顶部常量。

## 质量门（复用现有机器，别造新轮子）
管线（一个候选要**全过**才进登记簿）：
1. **placebo 分段**（`placebo_test.py`）：全样本 vs 现代段都显著 + recent_min_group_n 检验力够。
2. **跨族 FDR**（现有 FDR 脚本，Yekutieli）：在**整个候选家族**上算 q 值，q<0.10 才算。
3. **样本外**（`walk_forward.py` / CPCV）：样本外 AUC/收益显著 > 基率。
4. **Deflated Sharpe**（现有；若无则建）：按 N=家族大小缩水后仍 >0。
5. **conformal**（现有）：预测区间不离谱。
任一不过 → 进坟场，记**死在哪一门**（这本身就是诚实结果，复用 §4c/坟场口径）。

## 并入 + 度量
- **登记簿**（真规律）：append-only + hash 链（同 paper_ledger）。每行：候选 id · 家族 · 发现日 · 各门结果 · q 值 · 样本外指标 · Deflated Sharpe。
- **坟场**（假规律）：append-only。每行：候选 · 死因门 · 当时指标。
- **长期复测（度量·诚实 meta 检查）**：每轮对**已入登记簿**的规律重测样本外；**衰减（不再显著）→ 移回坟场，记"曾真、现已套利/失效 + 日期"**。这是整个系统最诚实的一环——承认"真规律也会死"。
- 账本损坏防护：复用 `repair_ledgers.py` 口径。

## 实现（增量，每步可验证 + 单审/双审）
1. `candidate_space.py`：枚举家族（N 固定常量 + 注释为何这些、为何不更多）。【双审：空间是否有界、有无暗藏自由度】
2. `quality_gate.py`：串现有门，对**整个家族**算 FDR + Deflated Sharpe，输出每候选「过/死在哪门 + q + DSR」。【双审：FDR 是否跨族、DSR 的 N 对不对】
3. `grow_registry.py`：把结果写 append-only 登记簿/坟场 + hash 封链；含**长期复测 + 衰减移回**逻辑。
4. run_all 注册（独立、可跑慢；非 light）。pytest：合成数据造"真信号"和"纯噪声各 N 个"，断言噪声组 FDR 后入选 ≈ N×q（不爆表）、真信号入选。
5. 前端：登记簿/坟场页加「自生长」区 + **诚实账单**（测了 N、期望假阳 N×α、入选 M、复测衰减 K）。

## 验收 / 审（最高规格）
- **双审必做**（Fable 建 + 全新 Opus/Codex）：审查重点 = **多重检验是否跨族、DSR 的 trials 数、有无 HARKing、候选空间有无暗藏无界自由度**。
- 合成数据单测：**纯噪声家族**跑完，入选数必须 ≈ 期望假阳率（证明门没漏）。这是上线的**硬门**。
- 全量 run_all + verify_output + 浏览器验证。

## 不做（红线）
- 不无界生成候选 / 不逐候选不更正 / 不看结果调门槛（HARKing）/ 不加新因子或模型。
- 不把"自生长"包装成"会赚钱的策略"——它只是**诚实地维护一份'什么是真规律'的活账本**，仍守"测风险不测方向"（除非某条进了出格区，另说）。
